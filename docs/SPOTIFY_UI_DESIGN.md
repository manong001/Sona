# Sona Spotify 风格 UI 设计规范

版本：1.0  
目标客户端：iOS 17+ / SwiftUI  
视觉基准：用户提供的 5 张 Spotify iOS 截图，原始尺寸均为 1320 × 2868 px，对应约 440 × 956 pt @3x。

## 1. 设计目标

Sona 保留私人曲库、无损播放、服务端歌单、收藏、历史和用户管理能力，界面采用 Spotify 式的信息密度与操作层级：深色背景、大标题、横向内容架、紧凑卡片、悬浮迷你播放器和底部导航。

“高度还原”指以下内容：

- 页面结构、留白、组件比例、层级、颜色和滚动方式接近参考图。
- 首页、搜索、音乐库、侧边账户抽屉和迷你播放器覆盖参考图的核心布局。
- 使用 Sona 的真实曲库和用户数据，不伪造 Spotify 推荐、播客、Premium 或社交内容。
- 使用 Sona 名称、图标和主题色，不复制 Spotify 商标或专属素材。

## 2. 信息架构

底部导航固定为四项：

| 标签 | 图标 | 主要内容 | 参考图映射 |
|---|---|---|---|
| 首页 | `house.fill` | 快捷入口、最近播放、收藏、歌单、专辑、艺人 | 图 1、图 2 |
| 搜索 | `magnifyingglass` | 全局搜索、曲目结果、内容分类卡 | 图 5 |
| 音乐库 | `books.vertical.fill` | 收藏、歌单、专辑、艺人切换与排序 | 图 4 |
| 设置 | `gearshape.fill` | 服务器、账户安全、用户管理、曲库扫描 | Sona 原功能 |

点击首页、搜索或音乐库左上角头像，打开账户侧边抽屉；抽屉覆盖屏幕约 88% 宽度，右侧保留暗色遮罩，结构参考图 3。

## 3. 设计令牌

### 3.1 颜色

| 名称 | 色值 | 用途 |
|---|---|---|
| `background` | `#121212` | 页面主背景 |
| `backgroundDeep` | `#080808` | 底栏与渐变底部 |
| `surface` | `#242424` | 快捷卡片、输入框、抽屉功能块 |
| `surfaceRaised` | `#2A2A27` | 迷你播放器 |
| `surfacePressed` | `#333333` | 按压、选中前景 |
| `primaryText` | `#FFFFFF` | 标题、主信息 |
| `secondaryText` | `#B3B3B3` | 副标题、说明、未选中图标 |
| `tertiaryText` | `#7C7C7C` | 禁用信息 |
| `accent` | `#1ED760` | Sona 绿色、激活状态、设备提示 |
| `divider` | `#2F2F2F` | 抽屉与设置分隔线 |
| `danger` | `#E91429` | 删除、停用、错误 |

所有页面强制深色外观。背景不使用系统 `Form` 默认灰色；需要列表时使用透明行背景和 `#121212` 容器。

### 3.2 字体

使用系统 SF Pro，中文由系统字体回退：

| 场景 | 字号 / 字重 |
|---|---|
| 页面标题 | 30–32 pt / Bold |
| 区块标题 | 22–24 pt / Bold |
| 卡片标题 | 15–16 pt / Semibold |
| 正文 | 15–16 pt / Regular |
| 元数据 | 13–14 pt / Regular |
| 底栏标签 | 11 pt / Medium |
| 迷你播放器标题 | 14 pt / Medium |

标题最多两行，曲目标题通常一行省略；元数据使用 `secondaryText`。

### 3.3 间距与圆角

- 页面水平边距：16 pt。
- 区块纵向间距：28–34 pt。
- 卡片内部间距：10–12 pt。
- 紧凑元素间距：6–8 pt。
- 快捷卡圆角：6 pt。
- 内容封面圆角：4–8 pt；艺人头像使用圆形。
- 搜索框圆角：8 pt。
- 筛选胶囊高度：34 pt，水平内边距 16 pt。
- 最小点击区域：44 × 44 pt。

## 4. 全局组件

### 4.1 用户头像

- 尺寸：40 × 40 pt；首页筛选栏中为 36 × 36 pt。
- 当前没有头像字段，使用用户名首字母与深绿渐变生成。
- 点击后从左侧弹出账户抽屉。

### 4.2 筛选胶囊

- 未选中背景：`#333333`，文字白色。
- 选中背景：`#1ED760`，文字 `#07150B`。
- 首页使用“全部 / 音乐”；音乐库使用“歌单 / 专辑 / 艺人”。
- 切换采用 0.2 秒 ease-in-out，内容不做夸张转场。

### 4.3 内容封面

- 歌单、专辑：方形，圆角 6 pt。
- 艺人：圆形。
- 无封面时显示 `surface` 背景、Sona 绿色音符和轻微渐变。
- 图片加载保持灰色占位，禁止布局跳动。

### 4.4 悬浮迷你播放器

- 位于底部标签栏上方，左右边距 8 pt，高度 64 pt。
- 背景：`surfaceRaised`，圆角 9 pt。
- 左侧封面：48 × 48 pt，圆角 5 pt。
- 中间：标题一行，艺人一行；设备/品质信息使用绿色。
- 右侧必须保留上一曲、播放/暂停、下一曲按钮；紧凑屏幕允许按钮间距缩小，但不能隐藏。
- 底部进度线：2 pt，轨道白色 25% 透明度，进度白色 70%。
- 整体左右滑动：左滑下一曲，右滑上一曲；点击信息区打开播放页。

### 4.5 底部导航

- 使用系统 `TabView` 保持辅助功能和安全区行为。
- 背景使用从透明到 `#080808` 的渐变，并保持迷你播放器独立悬浮。
- 激活项白色，未激活项 `secondaryText`。
- 图标约 24 pt，文字 11 pt。

## 5. 页面规范

### 5.1 首页

顶部固定内容：

1. 头像。
2. “全部 / 音乐”筛选胶囊。
3. 下方内容可纵向滚动，顶部在滚动时保持深色背景。

首屏快捷区域：

- 两列网格，列间距 8 pt，行间距 8 pt。
- 单卡高度 64 pt，左侧封面 64 × 64 pt，右侧标题 15 pt Semibold。
- 最多显示 6 个入口，按“收藏的歌曲、最近播放、最近歌单”顺序生成。

纵向区块：

- “最近播放”：横向滚动，展示最近历史，艺人用圆形，专辑/歌单用方形。
- “收藏的歌曲”：横向滚动，展示收藏曲目封面。
- “你的歌单”：横向滚动，展示用户服务端歌单。
- “浏览专辑”：从曲库按专辑名去重。
- “常听艺人”：从最近播放与曲库中按艺人去重。

横向卡宽 152–168 pt，封面宽等于卡宽；标题与副标题位于封面下方。屏幕右侧需要露出下一张卡的一部分，提示可横滑。

### 5.2 搜索

顶部：头像、32 pt “搜索”标题；右侧不放无功能的相机按钮。

搜索框：

- 高 52 pt，白色背景，黑色图标与文字。
- 未输入时提示“想听什么？”。
- 输入时实时过滤已加载曲库，支持歌曲、艺人和专辑。
- 有结果时立即切换为曲目列表；清空后恢复发现内容。

发现内容：

- “发现新内容”：横向展示派生分类，如无损、带歌词、已下载、收藏。
- “浏览全部”：两列彩色卡片，尺寸约 190 × 112 pt。
- 分类卡采用高饱和背景、左上角白色粗体标题、右下角旋转的方形封面。
- 点击分类进入对应过滤后的曲目列表；不展示无实际内容的卡片。

### 5.3 音乐库

顶部首行：头像、32 pt “音乐库”、搜索按钮、加号按钮。

- 搜索按钮在当前音乐库内展开搜索框。
- 加号打开“新建歌单”输入弹窗。
- 下一行是“歌单 / 专辑 / 艺人”筛选胶囊。
- 排序栏显示“最近播放”与网格/列表切换图标；本版默认列表。

列表内容：

- 歌单模式第一项固定为“收藏的歌曲”，使用紫绿渐变方形封面和白色心形。
- 后续为用户歌单，副标题格式“歌单 · 用户名”。
- 专辑模式：方形封面，副标题“专辑 · 艺人”。
- 艺人模式：圆形头像，副标题“艺人”。
- 单行图片 64 × 64 pt，行高 80 pt，文本与图片间距 14 pt。

### 5.4 账户侧边抽屉

- 宽度：屏幕宽度的 88%，最大 390 pt。
- 背景：`#202020`；右侧遮罩为黑色 55% 透明度。
- 顶部用户区域高约 118 pt：56 pt 头像、用户名、角色说明。
- 菜单行高 58 pt，图标 26 pt，文字 17 pt。
- 菜单项：最近播放、设置和隐私；管理员额外显示用户管理；底部显示退出登录。
- 点击遮罩或向左拖动关闭，动画 0.24 秒 ease-out。

不展示当前 Sona 不支持的“好友、消息、活动、添加账号”。

### 5.5 设置

保留现有服务器、曲库维护、账户安全、用户管理和关于功能，但改用同一深色令牌：

- 大标题 30 pt。
- 分组背景 `surface`，圆角 12 pt。
- 绿色用于可执行主操作，红色仅用于退出和删除。
- 普通用户隐藏曲库扫描与用户管理。

### 5.6 正在播放

- 顶部为向下关闭、居中“正在播放”、右侧更多按钮占位空间。
- 背景由当前封面主色无法稳定提取时，使用 `surface` 到黑色的纵向渐变。
- 封面宽度为屏幕宽度减 56 pt，1:1，圆角 8 pt。
- 标题 22 pt Bold，艺人 16 pt secondary。
- 收藏按钮与标题同一行右侧。
- 进度条使用细白线；时间 11 pt。
- 播放控制必须有随机、上一曲、播放/暂停、下一曲、循环。
- 底部保留离线下载和歌词入口。

## 6. 数据映射

| UI 内容 | Sona 数据来源 |
|---|---|
| 收藏的歌曲 | `PersonalStore.favoriteIDs` + `LibraryStore.tracks` |
| 最近播放 | `PersonalStore.history` |
| 用户歌单 | `PersonalStore.playlists` |
| 专辑 | `Track.album` 去重，封面取专辑第一首 |
| 艺人 | `Track.artist` 去重，封面取艺人第一首 |
| 搜索结果 | 标题、艺人、专辑本地过滤 |
| 无损分类 | FLAC、ALAC、WAV、AIFF 或 bit depth ≥ 16 |
| 带歌词分类 | `Track.hasLyrics == true` |
| 已下载分类 | `OfflineStore.downloadedIDs` |

## 7. 交互与状态

- 下拉刷新：首页刷新个人数据；音乐库刷新曲库与个人数据。
- 空状态仍保持 Spotify 式深色页面，不使用大面积系统灰色 `ContentUnavailableView`。
- 网络错误用底部短条提示，不遮挡迷你播放器。
- 创建、删除、收藏等操作成功后即时更新当前列表。
- 切换用户时清空个人数据并停止播放，防止跨账号残留。
- 迷你播放器始终通过 `safeAreaInset` 位于内容上方，不覆盖导航栏和列表最后一行。

## 8. 验收标准

在 440 × 956 pt 基准设备上检查：

1. 首页首屏能看到顶部筛选、2 列快捷卡、至少一个横向内容架和悬浮播放器。
2. 所有页面背景、卡片、文字颜色符合令牌，不出现系统默认浅色列表背景。
3. 搜索空态与结果态可切换，分类卡点击后显示真实曲目。
4. 音乐库三种筛选均使用正确封面形状和真实数据。
5. 账户抽屉宽度、遮罩、菜单行高和关闭手势接近参考图。
6. 迷你播放器同时显示上一曲、播放/暂停、下一曲，且左右滑动有效。
7. 播放页在小屏和大屏上不截断控制按钮，歌词与离线入口可用。
8. 管理员和普通用户看到的设置入口符合权限。
9. iOS 17 模拟器 Debug 与 iphoneos Release 编译通过。
10. 未签名 IPA 包含完整 Info.plist、AppIcon 和 arm64 可执行文件，并通过临时重签验证。
